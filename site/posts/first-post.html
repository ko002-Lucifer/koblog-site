使用 GitHub Pages 搭建个人博客

2026-07-03 · 作者: Student

前言

在当今互联网时代，拥有一个个人博客是分享知识、记录成长的重要方式。GitHub Pages 是 GitHub 提供的免费静态网站托管服务，配合 GitHub Actions 可以实现代码推送后自动部署，非常适合技术人员搭建个人博客。

什么是 GitHub Pages？

GitHub Pages 是一个静态网站托管服务，它直接从 GitHub 仓库中获取 HTML、CSS 和 JavaScript 文件，可以选择通过特定分支或者 GitHub Actions 工作流来发布网站。每个 GitHub 账户可以获得一个用户站点（username.github.io）和无限数量的项目站点（username.github.io/repo-name）。

核心步骤

1. 创建仓库

首先在 GitHub 上创建一个公开仓库。仓库名称可以任意命名（如 my-blog），这样网站地址就是 https://username.github.io/my-blog/。

2. 编写博客内容

使用 HTML、Markdown 或静态网站生成器（如 Hugo、Jekyll、VitePress 等）编写博客文章。本项目使用了简单的 HTML 页面来展示博客内容。

3. 配置 GitHub Actions

在仓库中创建 .github/workflows/deploy.yml 文件，配置自动部署工作流。每当代码推送到主分支时，GitHub Actions 会自动运行，将静态文件部署到 GitHub Pages。

工作流主要包含以下步骤：

  • 检出仓库代码（actions/checkout）
  • 配置 Pages（actions/configure-pages）
  • 上传静态文件（actions/upload-pages-artifact）
  • 部署到 Pages（actions/deploy-pages）

4. 启用 GitHub Pages

在仓库的 Settings → Pages 中，将 Source 设置为 GitHub Actions。这样部署工作流完成后，网站会自动更新。

技术栈选择

对于个人博客，有多种优秀的开源工具可以选择：

  • Hugo：Go 语言编写，构建速度极快
  • Jekyll：GitHub Pages 原生支持，Ruby 编写
  • VitePress：基于 Vue 和 Vite，适合文档和博客
  • MkDocs：Python 编写，适合项目文档
  • Hexo：Node.js 编写，主题丰富

本博客使用纯静态 HTML 方式搭建，配合 GitHub Actions 实现自动部署，简单高效。

总结

通过 GitHub Pages + GitHub Actions，我们可以零成本搭建一个自动部署的个人博客。整个流程利用了开源工具和免费服务，无需购买服务器，也无需复杂的运维操作。只要掌握 Git 和 Markdown/HTML 的基础知识，就能在短时间内拥有自己的技术博客。
